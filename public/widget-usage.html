üéØ Testimonial Widget - Implementation Guide

Transform your Twitter/X posts into beautiful testimonial widgets for your website. This guide shows you multiple ways to integrate our testimonial widget.

üöÄ Quick Start

The easiest way to add testimonials to your website:

Method 1: Simple Embed

Add this to your HTML:

<!-- Include Tailwind CSS and widget script -->
<script src="https://cdn.tailwindcss.com"></script>
<script src="https://your-domain.com/widget-embed.js"></script>

<!-- Create container with testimonial data -->
<div id="my-testimonials" data-testimonials='{
    "testimonials": [
        {
            "id": "1",
            "name": "John Doe",
            "title": "CEO at TechCorp",
            "avatar": "https://example.com/avatar.jpg",
            "content": "This tool is amazing! Highly recommend.",
            "rating": 5,
            "date": "Jan 15, 2024",
            "platform": "twitter"
        }
    ],
    "showNavigation": true,
    "autoSlide": true
}'></div>

Method 2: JavaScript Initialization

For more control over the widget:

<!-- Include Tailwind CSS and widget script -->
<script src="https://cdn.tailwindcss.com"></script>
<script src="https://your-domain.com/widget-embed.js"></script>

<!-- Create empty container -->
<div id="testimonial-widget"></div>

<script>
// Initialize widget with custom config
const widget = new TestimonialWidget({
    container: 'testimonial-widget',
    testimonials: [
        // Your testimonial data here
    ],
    showNavigation: true,
    showPagination: true,
    autoSlide: true,
    slideInterval: 5000,
    testimonialsPerPage: 3
});
</script>

Method 3: Load from API

Dynamically load testimonials from your server:

<script src="https://cdn.tailwindcss.com"></script>
<script src="https://your-domain.com/widget-embed.js"></script>

<div id="testimonial-widget"></div>

<script>
const widget = new TestimonialWidget({
    container: 'testimonial-widget',
    apiUrl: 'https://your-api.com/testimonials',
    showNavigation: true,
    autoSlide: true
});
</script>

‚öôÔ∏è Configuration Options

container

ID of the HTML element where the widget will be rendered

Default: 'testimonial-widget'

testimonials

Array of testimonial objects to display

Default: []

showNavigation

Show previous/next navigation buttons

Default: true

showPagination

Show pagination dots at the bottom

Default: true

autoSlide

Automatically advance slides

Default: true

slideInterval

Time between auto-slides (milliseconds)

Default: 5000

testimonialsPerPage

Number of testimonials to show per page

Default: 3

apiUrl

URL to fetch testimonials dynamically

Default: null

üìã Testimonial Data Format

Each testimonial should follow this structure:

{
    "id": "unique-identifier",
    "name": "Customer Name",
    "title": "Job Title (optional)",
    "handle": "@twitter_handle (optional)",
    "avatar": "https://example.com/avatar.jpg",
    "content": "The testimonial text content",
    "rating": 5, // 1-5 stars (optional)
    "date": "Jan 15, 2024",
    "platform": "twitter" // twitter, slack, linkedin, other
}

üé® Styling & Customization

The widget uses Tailwind CSS classes, which you can customize:

    @apply text-4xl text-purple-600;
}

/* Customize testimonial cards */
.testimonial-card {
    @apply bg-gradient-to-br from-blue-50 to-purple-50 border-purple-200;

/* Customize star colors */
.star.text-yellow-400 {
    @apply text-purple-400;
}

/* Customize navigation buttons */
.testimonial-nav button {
    @apply bg-purple-600 text-white hover:bg-purple-700;
}

üîß JavaScript API

Control the widget programmatically:

// Get widget instance

// Navigate slides
widget.nextSlide();
widget.previousSlide();
widget.goToPage(2);

// Update testimonials
widget.updateTestimonials(newTestimonialsArray);

// Control auto-sliding
widget.stopAutoSlide();
widget.startAutoSlide();

üì± Responsive Design

The widget automatically adapts to different screen sizes using Tailwind's responsive utilities:

  • Mobile: 1 testimonial per row (grid-cols-1)
  • Tablet: 2 testimonials per row (md:grid-cols-2)
  • Desktop: 3 testimonials per row (lg:grid-cols-3)

üí° Tips & Best Practices

  • Keep testimonial content concise (under 200 characters works best)
  • Use high-quality avatar images (minimum 80x80 pixels)
  • Include customer titles/companies for credibility
  • Mix different platforms (Twitter, Slack, LinkedIn) for variety
  • Update testimonials regularly to keep content fresh
  • Consider using 4-5 star ratings (perfect scores can seem fake)
  • Test the widget on different screen sizes and devices
  • Ensure images load properly with proper fallbacks
